Showcase game concept design doc: Rift Runner

One-line concept: Rift Runner is a 2D run-based action game where players dive into collapsing dimensions, recover world fragments, and reassemble a living city hub that changes gameplay each week.

Why this is stronger than “2D platforming roguelike”

  • Player fantasy: high-speed recovery missions through unstable worlds.
  • Core verbs: breach, route, recover, extract, rebuild.
  • Meta progression: recovered fragments physically rebuild the hub.
  • Weekly novelty: every drop adds a district fracture, mutator, boss, and upgrade.

Player fantasy and tone

  • You are a licensed runner in a city stitched from fractured realities.
  • Every run is high risk and has visible consequences in the hub.
  • Tone: high-contrast sci-fantasy, practical and gritty, no parody.
Readable movement Meaningful run choices Visible progress Weekly content drops

Core game loop

StepPlayer action
1Choose a contract in the hub (fracture type + risk profile).
2Enter the fracture run and clear combat-traversal rooms.
3Secure one primary fragment plus optional side fragments.
4Choose: extract now or push deeper for higher rewards.
5Return to hub and spend resources on permanent upgrades.
6Unlock new contracts and higher-risk modifiers.

Session structure (10-18 minute target)

StagePurpose
Entry zoneTeach current mutator safely.
Route splitDecision point: safer path vs higher reward path.
Elite gateBuild pressure and test readiness.
Boss arenaTwo-phase climax encounter.
ExtractionEscape sequence with outcome lock-in.

Hero V1 kit and enemy identity

  • Primary: pulse blade combo
  • Secondary: charge shot (armor break)
  • Mobility: dash with i-frames
  • Utility: tether pull (mobility + crowd control)
  • Ultimate: overdrive burst for clutch clears
  • Design rule: each weekly mutator must touch at least one kit component.
FactionBehavior identity
Signal CultZoning, delayed explosions, area denial.
Glass ChoirPrecision volleys, shields, reflections.
Vine CourtSwarm pressure, slows, stacking debuffs.
Ash FoundryHeavy armor, knockback hazards, heat pressure.

Progression and weekly drop framework

  • In-run economy: Charge (temporary buffs), Keys (risk rooms/rerolls).
  • Meta economy: Fragments (permanent currency), Relics (build archetypes).
  • District upgrades: forge, lab, transit, archive.
  • One fracture district variant
  • One featured boss
  • One global mutator
  • One new relic or district upgrade
  • One creator challenge contract

Art direction, MVP scope, and prompt strategy

Style target: cohesive stylized 2D with clear silhouettes and readable combat timing. Start with extensible placeholder assets (manifest-driven), then polish late. Menus/UI should stay lightweight until core loop quality is proven.

MVP checklist

  • Platform exploration loop with horizontal + vertical traversal
  • Collect rift key and complete run
  • Main menu + contract selection
  • Procedural level generation from explicit rules
  • Core movement, damage/death, hazards
  • Rift key progression into base rebuild + story beats

Parking lot

  • Powers and advanced abilities
  • Puzzles and secondary keys
  • Dynamic art assets per run

Prompt framing reference

Initial prompt (too broad):
"I want to create a cyberpunk style game called Rift Runners ... first phase is creating the main menu ... placeholder assets ... extensible components ..."

Improved prompt strategy:
Break work into discrete implementation phases and request a markdown plan with actionable prompts for each phase (core loop, contract select, procedural level rules, movement, hazards, progression, and polish).
